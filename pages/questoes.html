Questões

1. Para que serve o commit e qual a melhor forma de utilizá-lo?

O commit serve para registrar e salvar alterações em um sistema de controle de versão, como o Git. A melhor forma de utilizá-lo é fazendo commits frequentes, com mensagens descritivas e atômicas, mantendo um histórico organizado e facilitando o acompanhamento das mudanças no projeto.


2. Qual a diferença entre commit e push?

O commit registra as alterações no repositório local, enquanto o push envia essas alterações para o repositório remoto. O commit é uma ação local, enquanto o push atualiza o repositório compartilhado com as mudanças locais.


3. O que é tag?

Uma tag é uma referência estática e nomeada a um ponto específico na história de um repositório de controle de versão, como o Git. Ela é usada para marcar versões importantes do código, tornando mais fácil identificar e acessar esses pontos específicos.


4. O que é branch?

Uma branch é uma cópia independente do código-fonte em um repositório de controle de versão, como o Git. Ela permite que você trabalhe em novas funcionalidades ou correções sem afetar o código principal, facilitando o desenvolvimento paralelo e a posterior fusão das alterações.


5. Como resolver possíveis conflitos no Git?

Atualizanmdo seu repositório local com git pull para obter as alterações mais recentes do repositório remoto. Após isso, abra os arquivos com conflitos e edite-os manualmente para corrigir as diferenças. Depois de corrigir os conflitos, adicione os arquivos editados com git add. Então pode-se finalizar a resolução do conflito com git commit para criar um novo commit de resolução. Prosseguindo com git push para enviar as alterações resolvidas para o repositório remoto. Com isso, integramos as correções ao código principal e resolve os conflitos.
